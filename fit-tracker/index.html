Зависимости и ViewBinding

Подключаем Navigation, ViewModel, RecyclerView

Откройте build.gradle.kts (Module :app).

📄 build.gradle.kts (Module :app)
Gradle 
// В блоке android { } добавьте:
android {
    // ... существующий код ...

    buildFeatures {
        viewBinding = true
    }
}

// Замените блок dependencies:
dependencies {
    implementation(libs.androidx.core.ktx)
    implementation(libs.androidx.appcompat)
    implementation(libs.material)
    implementation(libs.androidx.activity)
    implementation(libs.androidx.constraintlayout)

    // Navigation
    implementation("androidx.navigation:navigation-fragment-ktx:2.7.7")
    implementation("androidx.navigation:navigation-ui-ktx:2.7.7")

    // ViewModel + LiveData
    implementation("androidx.lifecycle:lifecycle-viewmodel-ktx:2.7.0")
    implementation("androidx.lifecycle:lifecycle-livedata-ktx:2.7.0")

    // RecyclerView + CardView
    implementation("androidx.recyclerview:recyclerview:1.3.2")
    implementation("androidx.cardview:cardview:1.0.0")

    // ProgressBar (круговые диаграммы будем делать кастомно)

    testImplementation(libs.junit)
    androidTestImplementation(libs.androidx.junit)
    androidTestImplementation(libs.androidx.espresso.core)
}

🔍 Что даёт каждая библиотека?

  • Navigation — управление переходами. Забудьте про ручные FragmentTransaction!
  • ViewModel — хранит данные экрана при повороте. LiveData оповещает UI при изменении.
  • ViewBinding — типобезопасный доступ к View без findViewById. Генерирует класс-обёртку для каждого layout.
⚠️
После редактирования — жмите Sync Now в жёлтой полоске.

Модель Workout

Тренировка — основная сущность приложения

В пакете model → New → Kotlin Class/File → Data Class → Workout

📄 model/Workout.kt
Kotlin
package com.example.fittracker.model

data class Workout(
    val id: Int,
    val name: String,             // "Утренняя пробежка"
    val type: WorkoutType,        // CARDIO, STRENGTH...
    val durationMin: Int,         // длительность в минутах
    val calories: Int,            // сожжённые калории
    val exercises: List<Exercise>, // упражнения в тренировке
    val date: String,             // "2024-01-15"
    val difficulty: Difficulty = Difficulty.MEDIUM,
    val iconResId: Int,           // drawable-иконка
    var isCompleted: Boolean = false
)

enum class WorkoutType(val label: String, val emoji: String) {
    CARDIO("Кардио", "🏃"),
    STRENGTH("Силовая", "🏋️"),
    YOGA("Йога", "🧘"),
    STRETCHING("Растяжка", "🤸"),
    HIIT("HIIT", "⚡")
}

enum class Difficulty(val label: String) {
    EASY("Лёгкая"),
    MEDIUM("Средняя"),
    HARD("Тяжёлая")
}

🔍 Разбор кода

  • enum class WorkoutType — перечисление с параметрами. Каждый тип имеет человеко-читаемое имя и эмодзи. Это типобезопаснее, чем строки: компилятор не даст передать невалидный тип.
  • var isCompleted — мутабельное поле, потому что пользователь может отметить тренировку выполненной.
  • List<Exercise> — тренировка содержит список упражнений. Это отношение «один-ко-многим».

Модель UserProfile

Данные пользователя и его параметры
📄 model/UserProfile.kt
Kotlin
package com.example.fittracker.model

data class UserProfile(
    val name: String,
    val email: String,
    val age: Int = 25,
    val weight: Double = 70.0,     // кг
    val height: Int = 175,        // см
    val goalCalories: Int = 500,  // дневная цель ккал
    val goalWorkouts: Int = 4,    // тренировок в неделю
    val goalSteps: Int = 10000    // шагов в день
) {
    val bmi: Double
        get() = weight / ((height / 100.0) * (height / 100.0))

    val bmiCategory: String
        get() = when {
            bmi < 18.5 -> "Недостаточный вес"
            bmi < 25.0 -> "Нормальный вес"
            bmi < 30.0 -> "Избыточный вес"
            else -> "Ожирение"
        }
}

🔍 Вычисляемые свойства

  • val bmi: Double get() = ... — это вычисляемое свойство (computed property). Оно не хранится в памяти, а вычисляется при каждом обращении. Идеально для производных значений.
  • when { } — аналог switch/if-else. Идиоматичный Kotlin для выбора по диапазону.

FitRepository — хранилище данных

Синглтон с тренировками, статистикой и профилем

В пакете repository → New → Kotlin Class/File → Object → FitRepository

🏗️
Repository Pattern — единая точка доступа к данным. Фрагменты не знают, откуда берутся данные (БД, сеть, кэш). В будущем можно легко заменить захардкоженные данные на Room или Retrofit.
▶ 📄 FitRepository.kt — полный код
📄 repository/FitRepository.kt
Kotlin
package com.example.fittracker.repository

import androidx.lifecycle.MutableLiveData
import com.example.fittracker.R
import com.example.fittracker.model.*

object FitRepository {

    var currentUser: UserProfile? = null

    // Тренировки пользователя — LiveData для реактивного UI
    val workouts = MutableLiveData<MutableList<Workout>>(
        getDefaultWorkouts().toMutableList()
    )

    // Сегодняшняя статистика
    val todayCalories = MutableLiveData(0)
    val todaySteps = MutableLiveData(6540)
    val todayMinutes = MutableLiveData(0)
    val weeklyWorkouts = MutableLiveData(0)

    fun getDefaultWorkouts(): List<Workout> = listOf(
        Workout(1, "Утренняя пробежка", WorkoutType.CARDIO,
            30, 280, listOf(
                Exercise("Разминка", 1, 1, 0.0, 0, "Всё тело"),
                Exercise("Бег 5 км", 1, 1, 0.0, 0, "Ноги"),
                Exercise("Заминка", 1, 1, 0.0, 0, "Всё тело")
            ), "2024-12-20", Difficulty.EASY, R.drawable.ic_run),

        Workout(2, "Силовая: верх тела", WorkoutType.STRENGTH,
            45, 380, listOf(
                Exercise("Жим лёжа", 4, 10, 60.0, 90, "Грудь"),
                Exercise("Тяга в наклоне", 4, 10, 50.0, 90, "Спина"),
                Exercise("Жим стоя", 3, 12, 30.0, 60, "Плечи"),
                Exercise("Сгибания на бицепс", 3, 12, 14.0, 60, "Руки"),
                Exercise("Французский жим", 3, 12, 20.0, 60, "Руки")
            ), "2024-12-19", Difficulty.HARD, R.drawable.ic_strength),

        Workout(3, "Йога: баланс", WorkoutType.YOGA,
            40, 180, listOf(
                Exercise("Поза дерева", 3, 1, 0.0, 30, "Баланс"),
                Exercise("Воин I", 3, 1, 0.0, 30, "Ноги"),
                Exercise("Воин II", 3, 1, 0.0, 30, "Ноги"),
                Exercise("Планка", 3, 1, 0.0, 30, "Кор")
            ), "2024-12-18", Difficulty.EASY, R.drawable.ic_yoga),

        Workout(4, "HIIT: жиросжигание", WorkoutType.HIIT,
            25, 420, listOf(
                Exercise("Бёрпи", 4, 15, 0.0, 30, "Всё тело"),
                Exercise("Прыжки", 4, 20, 0.0, 30, "Ноги"),
                Exercise("Отжимания", 4, 12, 0.0, 30, "Грудь"),
                Exercise("Скалолаз", 4, 20, 0.0, 30, "Кор")
            ), "2024-12-17", Difficulty.HARD, R.drawable.ic_hiit),

        Workout(5, "Растяжка: всё тело", WorkoutType.STRETCHING,
            20, 90, listOf(
                Exercise("Наклоны", 2, 1, 0.0, 20, "Спина"),
                Exercise("Бабочка", 2, 1, 0.0, 20, "Ноги"),
                Exercise("Кошка-корова", 2, 1, 0.0, 20, "Спина")
            ), "2024-12-16", Difficulty.EASY, R.drawable.ic_stretch),

        Workout(6, "Силовая: ноги", WorkoutType.STRENGTH,
            50, 450, listOf(
                Exercise("Приседания", 4, 12, 80.0, 120, "Ноги"),
                Exercise("Жим ногами", 4, 12, 120.0, 90, "Ноги"),
                Exercise("Выпады", 3, 10, 20.0, 60, "Ноги"),
                Exercise("Подъёмы на носки", 4, 15, 40.0, 60, "Ноги")
            ), "2024-12-15", Difficulty.HARD, R.drawable.ic_strength)
    )

    fun getById(id: Int): Workout? =
        workouts.value?.find { it.id == id }

    fun getByType(type: WorkoutType): List<Workout> =
        workouts.value?.filter { it.type == type } ?: emptyList()

    fun completeWorkout(id: Int) {
        val list = workouts.value ?: return
        val w = list.find { it.id == id } ?: return
        w.isCompleted = true
        workouts.value = list // триггер LiveData

        // Обновляем статистику
        todayCalories.value = (todayCalories.value ?: 0) + w.calories
        todayMinutes.value = (todayMinutes.value ?: 0) + w.durationMin
        weeklyWorkouts.value = (weeklyWorkouts.value ?: 0) + 1
    }

    fun getTotalCalories(): Int =
        workouts.value?.filter { it.isCompleted }?.sumOf { it.calories } ?: 0

    fun getCompletedCount(): Int =
        workouts.value?.count { it.isCompleted } ?: 0

    // Данные для графика (последние 7 дней)
    fun getWeeklyData(): List<Pair<String, Int>> = listOf(
        "Пн" to 320, "Вт" to 0, "Ср" to 450,
        "Чт" to 280, "Пт" to 0, "Сб" to 520,
        "Вс" to (todayCalories.value ?: 0)
    )

    fun resetStats() {
        todayCalories.value = 0
        todaySteps.value = 6540
        todayMinutes.value = 0
        weeklyWorkouts.value = 0
        workouts.value = getDefaultWorkouts().toMutableList()
    }
}

🔍 Ключевые паттерны

  • MutableLiveData с начальным значением — при подписке observer сразу получит текущее значение.
  • workouts.value = list — обязательное присвоение для триггера обновления. Мутация содержимого списка без присвоения не триггерит observer.
  • completeWorkout() — обновляет и тренировку, и все связанные метрики атомарно.
  • getWeeklyData() — возвращает пары «день→калории» для графика.

Navigation Graphs

Два графа: основной (auth → main → detail) и вложенный (tabs)
🧭
Архитектура навигации FitTracker:
• nav_graph.xml: LoginFragment → RegisterFragment → MainFragment → WorkoutDetailFragment
• nav_main.xml: DashboardFragment, WorkoutsFragment, StatsFragment, ProfileFragment (вкладки)
▶ 📄 nav_graph.xml — основной
nav_graph.xml
<navigation xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:app="http://schemas.android.com/apk/res-auto"
    android:id="@+id/nav_graph"
    app:startDestination="@id/loginFragment">

    <fragment android:id="@+id/loginFragment"
        android:name="com.example.fittracker.ui.auth.LoginFragment">
        <action android:id="@+id/action_login_to_register"
            app:destination="@id/registerFragment" />
        <action android:id="@+id/action_login_to_main"
            app:destination="@id/mainFragment"
            app:popUpTo="@id/loginFragment"
            app:popUpToInclusive="true" />
    </fragment>

    <fragment android:id="@+id/registerFragment"
        android:name="com.example.fittracker.ui.auth.RegisterFragment">
        <action android:id="@+id/action_register_to_main"
            app:destination="@id/mainFragment"
            app:popUpTo="@id/loginFragment"
            app:popUpToInclusive="true" />
        <action android:id="@+id/action_register_to_login"
            app:destination="@id/loginFragment"
            app:popUpTo="@id/loginFragment"
            app:popUpToInclusive="true" />
    </fragment>

    <fragment android:id="@+id/mainFragment"
        android:name="com.example.fittracker.ui.MainFragment">
        <action android:id="@+id/action_main_to_detail"
            app:destination="@id/workoutDetailFragment" />
        <action android:id="@+id/action_main_to_login"
            app:destination="@id/loginFragment"
            app:popUpTo="@id/nav_graph"
            app:popUpToInclusive="true" />
    </fragment>

    <fragment android:id="@+id/workoutDetailFragment"
        android:name="com.example.fittracker.ui.detail.WorkoutDetailFragment">
        <argument android:name="workoutId" app:argType="integer" />
    </fragment>
</navigation>

LoginFragment

Вход с валидацией и имитацией аутентификации
ui/auth/LoginFragment.kt
Kotlin
package com.example.fittracker.ui.auth

import android.os.Bundle
import android.view.*
import android.widget.Toast
import androidx.fragment.app.Fragment
import androidx.navigation.fragment.findNavController
import com.example.fittracker.R
import com.example.fittracker.databinding.FragmentLoginBinding
import com.example.fittracker.model.UserProfile
import com.example.fittracker.repository.FitRepository

class LoginFragment : Fragment() {

    private var _binding: FragmentLoginBinding? = null
    private val binding get() = _binding!!

    override fun onCreateView(
        inflater: LayoutInflater, container: ViewGroup?,
        savedInstanceState: Bundle?
    ): View {
        _binding = FragmentLoginBinding.inflate(inflater, container, false)
        return binding.root
    }

    override fun onViewCreated(view: View, savedInstanceState: Bundle?) {
        super.onViewCreated(view, savedInstanceState)

        binding.btnLogin.setOnClickListener {
            val email = binding.etEmail.text.toString().trim()
            val pass = binding.etPassword.text.toString().trim()

            // Валидация
            if (email.isEmpty()) {
                binding.etEmail.error = "Введите email"
                return@setOnClickListener
            }
            if (pass.length < 4) {
                binding.etPassword.error = "Минимум 4 символа"
                return@setOnClickListener
            }

            // Имитация входа
            FitRepository.currentUser = UserProfile(
                name = email.substringBefore("@"),
                email = email
            )

            Toast.makeText(requireContext(),
                "Добро пожаловать! 💪", Toast.LENGTH_SHORT).show()

            findNavController().navigate(R.id.action_login_to_main)
        }

        binding.tvGoToRegister.setOnClickListener {
            findNavController().navigate(R.id.action_login_to_register)
        }
    }

    override fun onDestroyView() {
        super.onDestroyView()
        _binding = null // предотвращает утечку памяти
    }
}

🔍 Анатомия фрагмента

  • _binding (nullable) + binding (non-null getter) — стандартный паттерн ViewBinding во фрагменте. В onDestroyView обнуляем, потому что view может быть уничтожен раньше фрагмента.
  • return@setOnClickListener — возврат из лямбды (не из функции). Без @label return выйдет из onViewCreated.
  • findNavController().navigate() — безопасный переход по action ID из Navigation Graph.

// В onViewCreated:
val workoutId = arguments?.getInt("workoutId") ?: return
val workout = FitRepository.getById(workoutId) ?: return

// Заполнение данных
binding.tvName.text = workout.name

binding.rvExercises.adapter = ExerciseAdapter(workout.exercises)

binding.btnComplete.setOnClickListener {
    FitRepository.completeWorkout(workout.id)
    Toast.makeText(requireContext(),
        "Тренировка завершена! 🎉 +${workout.calories} ккал",

    findNavController().navigateUp()
}

// Кнопка назад
binding.btnBack.setOnClickListener {
    findNavController().navigateUp()
}

🔍 Цепочка обновлений

Когда пользователь нажимает «Завершить»:
1. FitRepository.completeWorkout() ставит isCompleted = true
2. Обновляет todayCalories, todayMinutes, weeklyWorkouts
3. LiveData триггерит обновление всех подписчиков
4. DashboardFragment автоматически обновляет метрики и ProgressBar-ы
5. StatsFragment пересчитывает итоги
Ни одна из этих обновлений не требует ручного вызова — это сила реактивного подхода с LiveData!
