2

Добавление зависимостей

Подключаем библиотеки навигации, ViewModel и UI-компонентов

Откройте файл build.gradle.kts (тот, что Module :app) и добавьте зависимости.

🔍 Зачем нужны эти зависимости?

  • Navigation Component — управляет переходами между экранами (фрагментами). Вместо ручного управления FragmentManager.
  • ViewModel + LiveData — хранит данные при повороте экрана. LiveData автоматически обновляет UI.
  • ViewBinding — безопасный доступ к элементам UI без findViewById.
  • RecyclerView — эффективный скроллируемый список (используется для карточек еды, корзины).
  • CardView — красивые карточки с тенями и скруглёнными углами.
📄 build.gradle.kts (Module :app)
Gradle 
// В блоке android { } добавьте в конец:
android {
    // ... то что уже есть ...

    buildFeatures {
        viewBinding = true
    }
}

// Блок dependencies { } замените полностью:
dependencies {
    implementation(libs.androidx.core.ktx)
    implementation(libs.androidx.appcompat)
    implementation(libs.material)
    implementation(libs.androidx.activity)
    implementation(libs.androidx.constraintlayout)

    // Navigation Component
    implementation("androidx.navigation:navigation-fragment-ktx:2.7.7")
    implementation("androidx.navigation:navigation-ui-ktx:2.7.7")

    // ViewModel и LiveData
    implementation("androidx.lifecycle:lifecycle-viewmodel-ktx:2.7.0")
    implementation("androidx.lifecycle:lifecycle-livedata-ktx:2.7.0")

    // CardView и RecyclerView
    implementation("androidx.cardview:cardview:1.0.0")
    implementation("androidx.recyclerview:recyclerview:1.3.2")

    // ViewPager2
    implementation("androidx.viewpager2:viewpager2:1.0.0")

    testImplementation(libs.junit)
    androidTestImplementation(libs.androidx.junit)
    androidTestImplementation(libs.androidx.espresso.core)
}

Структура папок

Организуем код по пакетам для удобства

В окне Project (слева, режим Android) раскройте app → java → com.example.foodexpress.

Для каждого пакета: ПКМ → New → Package → введите имя.

📁 com.example.foodexpress/ ├── 📁 adapter/ ← адаптеры для списков ├── 📁 model/ ← классы данных ├── 📁 ui/ │ ├── 📁 auth/ ← вход и регистрация │ ├── 📁 home/ ← главная страница │ ├── 📁 catalog/ ← каталог блюд │ ├── 📁 cart/ ← корзина │ ├── 📁 profile/ ← профиль пользователя │ └── 📁 detail/ ← детальная страница блюда ├── 📄 DataManager.kt ← синглтон-хранилище данных └── 📄 MainActivity.kt ← точка входа

🔍 Почему именно такая структура?

Разделение по feature-пакетам (auth, home, cart...) — это стандартный подход в Android-разработке. Каждая фича имеет свой фрагмент и layout. Так проще находить файлы и масштабировать проект.

Модель FoodItem

Класс данных для блюда

В пакете model → New → Kotlin Class/File → выберите Data Class → имя FoodItem

📄 model/FoodItem.kt
Kotlin 
package com.example.foodexpress.model

data class FoodItem(
    val id: Int,                // Уникальный ID блюда
    val name: String,            // Название: "Маргарита"
    val description: String,     // Описание блюда
    val price: Double,           // Цена в рублях
    val imageResId: Int,         // Ссылка на drawable-картинку
    val category: String,        // Категория: "Пицца", "Суши"
    val rating: Float = 4.5f,    // Рейтинг (по умолч. 4.5)
    val deliveryTime: String = "30-40 мин",  // Время доставки
    var quantity: Int = 0       // Кол-во в корзине (var!)
)

🔍 Разбор кода

  • data class — специальный класс Kotlin, автоматически генерирует equals(), hashCode(), toString(), copy(). Идеален для моделей.
  • val — неизменяемое свойство (read-only). var — изменяемое. Обратите внимание: quantity — это var, потому что количество в корзине меняется.
  • imageResId: Int — это ID ресурса из R.drawable.*. В реальном проекте здесь была бы URL-строка для загрузки из сети.
  • Значения по умолчанию (= 4.5f) позволяют не указывать параметр при создании объекта.

DataManager — хранилище данных

Синглтон, который хранит товары, корзину и данные пользователя

В корневом пакете com.example.foodexpress → New → Kotlin Class/File → Object → имя DataManager

💡
Object в Kotlin — это синглтон. Один экземпляр на всё приложение. Все фрагменты будут обращаться к одному и тому же DataManager, что позволяет разделять данные корзины между экранами.
▶ 📄 DataManager.kt — полный код (нажмите чтобы свернуть/развернуть)
📄 DataManager.kt
Kotlin 
package com.example.foodexpress

import androidx.lifecycle.MutableLiveData
import com.example.foodexpress.model.FoodItem
import com.example.foodexpress.model.User

object DataManager {

    // Текущий авторизованный пользователь
    var currentUser: User? = null

    // Корзина — LiveData, чтобы UI обновлялся автоматически
    val cartItems = MutableLiveData<MutableList<FoodItem>>(mutableListOf())

    // Все блюда — имитация ответа от сервера
    fun getAllFood(): List<FoodItem> {
        return listOf(
            FoodItem(1, "Маргарита",
                "Классическая пицца с томатным соусом, моцареллой и базиликом.",
                450.0, R.drawable.placeholder_food_1, "Пицца", 4.8f, "25-35 мин"),
            FoodItem(2, "Пепперони",
                "Острая пицца с пепперони, моцареллой и томатным соусом.",
                520.0, R.drawable.placeholder_food_1, "Пицца", 4.9f, "25-35 мин"),
            FoodItem(3, "Четыре сыра",
                "Моцарелла, горгонзола, пармезан и эмменталь.",
                580.0, R.drawable.placeholder_food_1, "Пицца", 4.7f, "30-40 мин"),
            FoodItem(4, "Филадельфия",
                "Ролл со сливочным сыром, лососем и огурцом. 8 шт.",
                390.0, R.drawable.placeholder_food_2, "Суши", 4.6f, "40-50 мин"),
            FoodItem(5, "Калифорния",
                "Ролл с крабом, авокадо и тобико. 8 шт.",
                420.0, R.drawable.placeholder_food_2, "Суши", 4.5f, "40-50 мин"),
            FoodItem(6, "Дракон",
                "Запечённый ролл с угрём и авокадо. 8 шт.",
                490.0, R.drawable.placeholder_food_2, "Суши", 4.8f, "45-55 мин"),
            FoodItem(7, "Чизбургер",
                "Говяжья котлета, чеддер, огурцы, горчица.",
                280.0, R.drawable.placeholder_food_3, "Бургеры", 4.4f, "20-30 мин"),
            FoodItem(8, "Двойной бургер",
                "Две котлеты, двойной сыр, бекон, фирменный соус.",
                420.0, R.drawable.placeholder_food_3, "Бургеры", 4.7f, "20-30 мин"),
            FoodItem(9, "Тирамису",
                "Итальянский десерт с маскарпоне и кофе.",
                290.0, R.drawable.placeholder_food_4, "Десерты", 4.9f, "15-20 мин"),
            FoodItem(10, "Чизкейк",
                "Нежный Нью-Йорк чизкейк с ягодным соусом.",
                320.0, R.drawable.placeholder_food_4, "Десерты", 4.8f, "15-20 мин")
        )
    }

    fun getCategories(): List<Pair<String, String>> {
        return listOf(
            "🍕" to "Пицца",
            "🍣" to "Суши",
            "🍔" to "Бургеры",
            "🍰" to "Десерты"
        )
    }

    fun getFoodById(id: Int): FoodItem? =
        getAllFood().find { it.id == id }

    fun getFoodByCategory(category: String): List<FoodItem> =
        getAllFood().filter { it.category == category }

    // ===== Операции с корзиной =====

    fun addToCart(food: FoodItem) {
        val cart = cartItems.value ?: mutableListOf()
        val existing = cart.find { it.id == food.id }
        if (existing != null) {
            existing.quantity++
        } else {
            cart.add(food.copy(quantity = 1))
        }
        cartItems.value = cart
    }

    fun removeFromCart(foodId: Int) {
        val cart = cartItems.value ?: return
        cart.removeAll { it.id == foodId }
        cartItems.value = cart
    }

    fun updateQuantity(foodId: Int, quantity: Int) {
        val cart = cartItems.value ?: return
        val item = cart.find { it.id == foodId } ?: return
        if (quantity <= 0) cart.remove(item)
        else item.quantity = quantity
        cartItems.value = cart
    }

    fun getCartTotal(): Double =
        cartItems.value?.sumOf { it.price * it.quantity } ?: 0.0

    fun getCartCount(): Int =
        cartItems.value?.sumOf { it.quantity } ?: 0

    fun clearCart() {
        cartItems.value = mutableListOf()
    }
}

🔍 Ключевые концепции

  • MutableLiveData — обёртка над данными, при изменении .value все подписчики (фрагменты) получают уведомление и обновляют UI автоматически.
  • cartItems.value = cart — именно присвоение нового значения триггерит обновление. Просто cart.add() без присвоения не сработает.
  • food.copy(quantity = 1) — создаёт копию data class с изменённым полем. Оригинал не меняется.
  • ?.sumOf { } — безопасный вызов + суммирование. Если список null, вернётся 0.

Drawable-ресурсы (фоны и формы)

XML-формы для кнопок, полей ввода и карточек

Для каждого файла: ПКМ на res/drawable → New → Drawable Resource File → введите имя.

💡
Drawable XML-файлы описывают формы без растровой графики. Они легковесны, масштабируются на любой экран и легко меняются.
▶ 📄 bg_button_primary.xml — фон основной кнопки
bg_button_primary.xml 
<shape xmlns:android="http://schemas.android.com/apk/res/android"
    android:shape="rectangle">
    <solid android:color="@color/primary" />
    <corners android:radius="@dimen/corner_radius" />
</shape>
▶ 📄 bg_edit_text.xml — фон поля ввода
bg_edit_text.xml 
<shape xmlns:android="http://schemas.android.com/apk/res/android"
    android:shape="rectangle">
    <solid android:color="@color/surface" />
    <stroke android:width="1dp" android:color="@color/divider" />
    <corners android:radius="@dimen/corner_radius" />
    <padding android:left="16dp" android:right="16dp"
        android:top="4dp" android:bottom="4dp" />
</shape>
▶ 📄 Ещё 6 drawable-файлов (bg_card, bg_category, bg_badge, bg_circle_button, bg_profile_header, bg_profile_item)

Создайте эти файлы аналогично. Полный код каждого файла приведён в текстовой версии инструкции в предыдущем сообщении. Основные паттерны:

  • <shape> с <solid> и <corners> — скруглённый прямоугольник
  • <shape android:shape="oval"> — круг (для бейджей и кнопок)
  • <gradient> — градиентный фон (для шапки профиля)
  • <ripple> — эффект нажатия (для пунктов меню)

Также создайте 4 файла placeholder_food_1..4.xml — это цветные заглушки вместо реальных фотографий еды (градиенты разных цветов).

10

Векторные иконки

18 иконок для навигации, действий и меню

Все иконки — XML-файлы с <vector>. Создаются в res/drawable/. Полный код каждой иконки приведён в основной инструкции (ic_home, ic_catalog, ic_cart, ic_person, ic_search, ic_star, ic_time, ic_add, ic_remove, ic_delete, ic_arrow_back и др.).

🔍 Почему Vector Drawable?

Векторные иконки не теряют качество при масштабировании. Один файл работает на всех плотностях экрана (hdpi, xhdpi, xxhdpi...), в отличие от PNG, где нужно 4-5 версий.

Navigation Graph

Граф переходов между экранами

Создайте папку navigation: ПКМ на res → New → Android Resource Directory → Resource type: navigation.

🏗️
Архитектура навигации: У нас два навигационных графа:
• nav_graph.xml — верхний уровень: Login → Register → Main → Detail
• nav_main.xml — вложенный (внутри Main): Home, Catalog, Cart, Profile
Такая структура позволяет скрывать Bottom Navigation на экране логина и деталей.
▶ 📄 nav_graph.xml — основной граф

13

Activity и экраны авторизации

activity_main.xml, fragment_login.xml, fragment_register.xml

Полный код всех layout-файлов приведён в текстовой инструкции выше. Ключевые моменты:

🔍 Структура layout-ов

Main Fragment и Home

fragment_main.xml с BottomNavigation, fragment_home.xml

🔍 Вложенная навигация

fragment_main.xml содержит вложенный NavHostFragment + BottomNavigationView. Это позволяет переключать Home/Catalog/Cart/Profile внутри Main, сохраняя нижнюю панель.

Каталог и Корзина

fragment_catalog.xml, fragment_cart.xml

🔍 Паттерн «Пустое состояние»

В fragment_cart.xml есть два блока: emptyCartLayout (пусто) и rvCart + checkoutLayout (есть товары). Фрагмент переключает видимость через View.VISIBLE / View.GONE.

16

Профиль и Детали блюда

fragment_profile.xml, fragment_detail.xml

🔍 include в layout

fragment_profile.xml использует <include> для повторяющихся пунктов меню. Каждый include получает уникальный android:id, что позволяет обращаться к нему через ViewBinding.

17

Элементы списков

item_category.xml, item_food.xml, item_cart.xml, item_filter.xml, item_profile_menu.xml

🔍 Item layout-ы для RecyclerView

Каждый элемент списка — отдельный XML-файл. RecyclerView переиспользует эти «шаблоны» при скролле, создавая только видимые элементы. Это намного эффективнее, чем ListView или динамическое создание View.

6 Адаптеры

18

CategoryAdapter

Горизонтальный список категорий с эмодзи

🔍 Паттерн RecyclerView.Adapter

  • onCreateViewHolder — создаёт View из XML-макета (вызывается редко)
  • onBindViewHolder — заполняет данными (вызывается при скролле)
  • getItemCount — сколько элементов в списке
  • Лямбда onClick передаётся извне — адаптер не знает, что делать при клике, это решает фрагмент (Separation of Concerns)
19

FoodAdapter

Карточки блюд с кнопкой «В корзину»

Принимает два callback-а: onItemClick (переход к деталям) и onAddToCart (добавить в корзину). Метод updateList() позволяет обновить данные при фильтрации.

20

CartAdapter

Элементы корзины с +/- и удалением

Callback onQuantityChanged(foodId, newQuantity) вызывается при нажатии +/-, а onDelete(foodId) — при удалении. Адаптер не управляет данными напрямую — только уведомляет фрагмент.

21

FilterAdapter

Фильтры категорий с выделением активного

Хранит selectedPosition и меняет фон выбранного элемента. При клике вызывает notifyItemChanged для старого и нового элемента — перерисовывает только их.

7 Фрагменты (логика экранов)

22

LoginFragment

Экран входа с валидацией

🔍 Паттерн ViewBinding во фрагменте

  • _binding — nullable, обнуляется в onDestroyView (предотвращает утечки памяти)
  • binding — non-null геттер для удобства, бросит исключение если view уже уничтожен
  • findNavController().navigate(R.id.action_...) — переход по заранее определённому action из Navigation Graph
  • Валидация: проверяем пустые поля и минимальную длину пароля перед переходом
23

RegisterFragment

Экран регистрации

Аналогичен LoginFragment, но с дополнительным полем имени. После регистрации сохраняет User в DataManager и переходит на MainFragment.

24

MainFragment — контейнер с Bottom Navigation

Связывает BottomNavigationView с вложенным NavHostFragment

🔍 Ключевой момент

  • childFragmentManager — менеджер вложенных фрагментов. Мы ищем NavHostFragment внутри MainFragment.
  • setupWithNavController() — магическая строка, которая связывает клики по нижнему меню с переходами в навигационном графе.
  • getOrCreateBadge() — создаёт бейдж (красный кружок с числом) на иконке корзины. Обновляется через LiveData.
25

HomeFragment

Главная страница с категориями и популярными блюдами

🔍 Навигация из вложенного фрагмента

HomeFragment вложен в MainFragment, который вложен в Activity. Чтобы перейти на DetailFragment (который в основном nav_graph), нужно использовать Navigation.findNavController(requireActivity(), R.id.nav_host_fragment) — это находит «верхний» навигатор.

26

CatalogFragment

Каталог с фильтрацией по категориям

Два RecyclerView: горизонтальный (фильтры) и вертикальный (список блюд). При клике на фильтр вызывается foodAdapter.updateList() с отфильтрованными данными.

27

CartFragment

Корзина с LiveData-обновлениями

🔍 Реактивность через LiveData

DataManager.cartItems.observe(viewLifecycleOwner) — подписка на изменения корзины. Когда любой фрагмент вызывает addToCart() или updateQuantity(), CartFragment автоматически обновляет UI: список, итоговую сумму, видимость «пустой корзины».

viewLifecycleOwner — подписка привязана к жизненному циклу view фрагмента. При уничтожении view подписка снимается автоматически (нет утечек).

28

ProfileFragment

Профиль с меню настроек и кнопкой выхода

Отображает данные из DataManager.currentUser. Пункты меню настраиваются через setupMenuItem(). Кнопка «Выйти» очищает данные и возвращает на LoginFragment.

29

DetailFragment

Детальная страница блюда

🔍 Передача данных между фрагментами

arguments?.getInt("foodId") — получаем ID блюда, переданный через Bundle при навигации. По нему загружаем данные из DataManager. В реальном проекте здесь был бы запрос к серверу.
